# 连锁超市云端协同系统

面向连锁零售与商超场景的云端协同管理平台，覆盖总部、总仓与门店之间的采购、调度、收货、盘点、货架管理与数据汇总。

## 项目简介

传统连锁零售企业往往依赖门店本地系统处理日常业务，导致总部难以及时掌握库存、采购、调拨和执行状态。本项目的目标是在不打断现有业务账本的前提下，建设一套云端协同平台，用于统一审批、下发、追踪与汇总多门店业务。

系统定位不是简单的单店进销存，而是服务于多门店、多仓、多角色协同的网络版管理平台。它适用于总部集中管控、区域仓配送、门店补货、跨店调拨、PDA 现场作业和经营分析等场景。

## 核心能力

- 基础资料管理：门店、商品、供应商、客户、会员、品牌、分类、仓库、单位等
- 补货与采购：补货申请、采购订单、采购入库、采购退货
- 调度中心：配送、收货、库存调拨、跨店调拨、调价、成本调整、促销下发
- PDA 作业：扫码收货、上架、巡检、盘点、货架任务执行
- 数据分析：销售、采购、库存、退货、趋势分析与经营看板
- 权限与隔离：按组织、角色、门店进行菜单权限和业务数据隔离

## 适用场景

- 连锁超市、便利店等多门店零售业态
- 总部统一管理商品、供应商、价格与促销策略
- 总仓向门店配送，或门店之间进行库存调拨
- 使用 PDA 进行一线收货、盘点、货架巡检与作业追踪
- 需要将门店现场数据汇总到云端进行分析和协同决策

## 架构概览

系统采用“云端协同平台 + 站点本地业务系统 + 同步中间层”的思路：

- 云端平台负责审批、下发、追踪、汇总和可视化
- 门店/仓库现场系统负责真实业务执行与落账
- 同步中间层负责站点与云端之间的数据交换和状态回传

这种架构既能满足总部统一协同，也便于逐步推进传统本地系统向云端演进。

## 阿里云部署用途

本项目计划部署在云服务器环境中，用于测试与逐步投入业务使用，典型资源包括：

- ECS：部署前后端服务与任务程序
- MySQL：存储云端协同数据与业务汇总数据
- Redis：缓存、会话与并发控制
- 对象存储或日志服务：存放附件、导入导出文件与运行日志
- 安全网络组件：用于连接各站点与云端环境
